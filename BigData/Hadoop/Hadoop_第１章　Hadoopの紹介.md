# 分散大規模データ処理システム -- Hadoop-1

## 第１章　Hadoopの紹介

### 第１節　Hadoopとは

　　HadoopはApache基金会によって開発された分散システムの基盤となるソフトウェアフレームワークです。これはストレージシステムと計算フレームワークを組み合わせたもので、大規模データについて保存と処理の問題を解決することを目的としている。

**Hadoopではどのような問題を解決した**

海量データの保存：HDFS（Hadoop Distributed File System）は高い耐障害性を備え、低コストのハードウェア上でのデプロイメントを目指して設計されています。また、大規模なデータセットを持つアプリケーションに適した高いスループットを提供し、データへのアクセスを高速化します。HDFSはDataNodeプロセスを実行するn台のマシンと、NameNodeプロセス（およびスタンバイ）を実行する1台のマシンで構成されています。各DataNodeはデータの一部を管理し、NameNodeはHDFSクラスタ全体の情報（メタデータを含む）を管理します。

1. リソースの管理、スケジューリング、割り当て

Apache Hadoop YARN（Yet Another Resource Negotiator、別のリソースコーディネーター）は、新しいHadoopリソースマネージャであり、汎用リソース管理システムおよびスケジューリングプラットフォームです。これは、上層のアプリケーションに一元化されたリソース管理とスケジューリングを提供します。YARNの導入により、クラスタの利用率、リソースの統合管理、データの共有などの面で大きな利点がもたらされました。

Hadoopは大規模なコンピュータクラスターに対して信頼性が高く、スケーラブルな計算とストレージのアプリケーションレイヤのサポートを提供します。これにより、単純なプログラミングモデルを使用して、コンピュータクラスター全体で大規模なデータセットを分散処理することが可能になります。また、単一のコンピュータから数千台のコンピュータにわたるスケールアップがサポートされています
